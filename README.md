# ATLAS STONE

Одностраничный сайт-каталог натурального камня и услуг по изготовлению и монтажу изделий. Проект написан на React, TypeScript и Vite.

## Запуск

Требуется Node.js 20.19+ или 22.12+.

```bash
npm ci
npm run dev
```

Локальный адрес по умолчанию: `http://localhost:5173`.

Для проверки отправки форм скопируйте `.env.example` в `.env`, укажите URL входящего вебхука Битрикс24 и
запустите API во втором терминале:

```bash
npm run dev:api
```

Vite перенаправляет запросы `/api` на `http://localhost:8787`. URL вебхука хранится только в `.env` и не должен
попадать в Git или в переменные с префиксом `VITE_`.

## Проверки

```bash
npm run check
```

Команда последовательно запускает линтер, автоматические тесты и production-сборку. Отдельно доступны `npm run lint`, `npm run test` и `npm run build`.

## Маршрутизация и публикация

Публичные адреса сайта: `/`, `/catalog`, `/services` и `/stone/:id`. Хостинг должен возвращать `index.html` для неизвестных путей, чтобы прямое открытие страниц работало как SPA. Для Netlify-совместимых платформ правило уже добавлено в `public/_redirects`; на другом сервере настройте эквивалентный rewrite.

Каталог загружается отдельным модулем только при переходе в каталог, карточку камня или при запуске подбора на главной. Это сохраняет лёгкую первоначальную загрузку главной страницы.

## Формы и production-запуск

Сервер `server/index.mjs` раздает production-сборку и принимает `POST /api/leads`. Заявки проверяются, получают
уникальный номер и создают лид в Битрикс24. Поддерживаются PDF, DWG, JPG и PNG: до 10 МБ на
файл и до 20 МБ суммарно. Перед публикацией выполните:

```bash
npm run build
npm start
```

Обязательные server-only переменные перечислены в `.env.example`. Входящий вебхук должен иметь права на CRM. Ответственный за лида сайтом не
передаётся и определяется настройками Битрикс24. Файлы попадают в множественное поле лида
«Файлы с сайта», которое интеграция создаёт при первой заявке с вложениями. Если поле создано вручную, его код можно указать в
`BITRIX24_FILE_FIELD`.

## Яндекс.Метрика

Укажите публичный номер счетчика в `VITE_YANDEX_METRIKA_ID` и пересоберите сайт. Пока переменная пуста, Метрика не
загружается. При заданном ID счетчик загружается только после выбора посетителем «Разрешить аналитику»; Вебвизор
отключен. Успешные формы отправляют цели `product_quote_sent`, `stone_quote_sent`, `project_quote_sent` и
`catalog_request_sent` без персональных данных.

## Персональные данные

На сайте опубликованы отдельные политика обработки персональных данных, согласие и политика cookies. Факт согласия,
версия документов, источник, дата и параметры заявки включаются в карточку лида. До запуска заказчик должен
утвердить тексты, указать фактического российского хостинг-провайдера во внутренних документах и проверить сведения в
реестре операторов персональных данных.
